### 8.2 沟通模板 ⭐NEW

#### 模板1：日常任务

```
【任务目标】：[一句话描述要做什么]
【期望结果】：[具体、可衡量的标准]
【当前问题】：(如有) [描述现状]
【禁止事项】：[哪些不能做]
【强制要求】：
  1. [要求1]
  2. [要求2]
【确认方式】：[如：复述需求后我再确认]
```

**示例**：
```
【任务目标】：修改卡片渐变背景
【期望结果】：1.渐变从左到右：深蓝→浅蓝；2.卡片占屏80%；3.标题24px
【当前问题】：目前没有渐变效果
【禁止事项】：不修改底部导航、不添加新功能
【强制要求】：1.用Context7查Tailwind语法；2.修改前git commit
【确认方式】：先给我看CSS代码，我确认后再运行
```

#### 模板2：紧急修复

```
【问题描述】：[什么出了问题]
【影响范围】：[如：部署失败/功能异常]
【紧急程度】：高/中/低
【期望时限】：[如：2小时内]
【禁止操作】：[如：不要改其他功能]
【处理流程】：定位原因 → 提出方案 → 你确认 → 执行
```

#### 模板3：需求讨论

```
【背景说明】：[为什么需要这个功能]
【初步想法】：[你的大致方向]
【需要AI做】：[希望AI提供什么帮助]
【你的顾虑】：[担心的问题]
【讨论方式】：[如：先聊方案，达成一致后再动手]
```

### 8.3 协作规则建议

```markdown
## 协作规则 [可复制使用]

### 代码相关
1. 任何修改前，必须先读取相关代码文件
2. 涉及部署，先查官方文档（Context7 MCP）
3. 重大修改前创建 git commit 快照

### 沟通相关
4. 使用标准化模板沟通
5. 先确认需求，再动手
6. 先给代码预览，再运行

### 反馈相关
7. 指出问题时，给出具体错误信息
8. 不满意时，说明期望结果
9. 允许回退，但需说明原因

### 优先级
10. 能用AI处理 → 优先AI，脚本仅传参
11. 质量优先 → 不接受"差不多"
12. 可回退 → 每次大改前备份
```
